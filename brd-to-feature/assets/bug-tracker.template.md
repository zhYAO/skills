## 1. Bug 状态说明

| 状态 | 含义 | 进入条件 | 离开条件 |
|------|------|----------|----------|
| `open` | 已发现，未修复 | bug 首次登记 | 开始修复 → `in_progress` |
| `in_progress` | 正在修复 | 开始改代码 | 修复完成 → `fixed` / 失败回退 → `open` |
| `fixed` | 已修复代码，等待验证 | 修复完成 | 回归通过 → `verified` / 验证失败 → `in_progress` |
| `verified` | 回归测试通过 | 自动回归全绿 | 用户验收 → `closed` / 验收失败 → `in_progress` |
| `closed` | 已关闭（验收 / 上线） | 用户确认 | （终态） |
| `wontfix` | 不修复 | 用户明确 | （终态）需注明原因 |

状态机：

```
        ┌──────────┐
        │  open    │ ← 发现 bug
        └────┬─────┘
             ↓
        ┌──────────┐
        │ in_progress │ ← 开始修复
        └────┬─────┘
             ↓
        ┌──────────┐
        │   fixed  │ ← 修复完成，等待验证
        └────┬─────┘
             ↓（回归通过 + 用户验收）
        ┌──────────┐
        │ verified │ ← 自动回归通过
        └────┬─────┘
             ↓
        ┌──────────┐
        │  closed  │ ← 上线 / 用户确认
        └──────────┘

（任何状态都可 → wontfix；fixed / verified 失败则回到 in_progress）
```

### 2.1 严重分级说明

- 🔴 **阻塞**：核心功能不可用 / 数据丢失 / 安全漏洞 / 流程走不通

- 🟡 **一般**：次要功能 / 边界 case / 文案 / UI 细节
- 🟢 **提示**：建议性改进 / 代码气味 / 不影响功能
